В чек лист включим следующие блоки для проверки:

1. Управление доступом.
   Эта важнейшая часть для обеспечения безопасности и целостности данных.
   С учетом наличия следующих проблем, необходимо включить блок в чек лист:
    - На текущий момент в компании есть несколько точек регистрации нового клиента, координации между этими точками нет,
      что затрудняет управление доступом.
    - Внутренние потоки данных между IT-продуктами контролируются не системно.
    - Со стороны систем обработки данных нет ограничений на доступ к данным.
    - Системы PropDevelopment используют API управляющих компаний без единых политик по безопасности.
    - Некоторые контракты позволяют внешним пользователям выполнять действия и получать данные, к которым у них не
      должно
      быть доступа
2. Безопасность данных.
   Есть явные проблемы с безопасностью данных:
    - Системы PropDevelopment используют API управляющих компаний без системного контроля за передаваемыми данными.
    - Контракты API содержат категории данных, которые явно предоставляют персональные данные.
3. Защита инфраструктуры.
   Этот блок необходимо добавить, так как в компании используются всевозможные компоненты инфраструктуры для работы
   автоматизированных систем:

- собственные сервера
- гибрид
- облачные ресурсы
- приватное облако, имеется собственный ЦОД и используются мощности, арендованные у облачного провайдера
  А так как в компании проводятся платежи, происходит передача финансовой и конфиденциальной информации,
  то необходимо осуществить соответствующие проверки, чтобы обеспечить приемлемый уровень безопасности инфраструктуры.

4. Управление инцидентами.
   Компания хочет избежать утечек данных и других критических инцидентов, которые могут повлиять на репутацию и
   финансовые потери.
   Для этого необходимо оперативно реагировать на инциденты и предотвращать их последствия.
5. Так как клиенты и сотрудники могут использовать мобильные устройства для подключения к сервисам компании,
   а системы развернуты как на собственных серверах, так и в облаке, то дополнительно добавим в чек лист следующие
   специфические аспекты:

- Мобильные устройства и удалённый доступ
- Локальная инфраструктура
- Облачная инфраструктура
